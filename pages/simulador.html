Visão Geral

O Simulador de Elementos de um Sistema Operacional é uma plataforma educacional e de testes construída para demonstrar na prática o comportamento de conceitos clássicos de Sistemas Operacionais. Com o frontend desenvolvido em React por outros integrantes do projeto, minha responsabilidade foi arquitetar e construir todo o motor por trás do simulador, focando em segurança, escalabilidade e comunicação de baixo nível.

A arquitetura backend foi desenvolvida em Java utilizando Spring Boot. O grande desafio técnico foi criar um sistema capaz de receber comandos da interface e executar algoritmos nativos em C de forma isolada. Para isso, configurei e integrei o backend ao Docker, permitindo a execução segura de testes rigorosos, como simulações de processos CPU-Bound e IO-Bound dentro de containers.

Principais Aspectos Técnicos

Containerização com Docker

Configuração e integração do backend com Docker para executar códigos nativos em C de forma isolada. Garante um ambiente seguro (sandbox) para rodar processos sem comprometer o host.

WebSockets em Tempo Real

Implementação de comunicação bidirecional via WebSockets para a simulação do problema clássico Produtor-Consumidor, fazendo o streaming contínuo dos resultados de execução para o frontend (React).

Simulação de Processos de SO

Desenvolvimento e controle de testes clássicos de Sistemas Operacionais. A API é capaz de acionar e monitorar algoritmos em C que demonstram certos aspectos de Sisetmas Operacionais em geral.

Arquitetura Spring Boot

Construção de um servidor robusto em Java que atua como o cérebro do sistema, orquestrando perfeitamente a comunicação entre as requisições web do usuário e a execução em baixo nível.
