Visão Geral

O projeto de Gestão de Limpeza Urbana foi desenvolvido para consolidar conhecimentos em modelagem relacional e persistência de dados num ambiente profissional. A aplicação utiliza o ecossistema Java Spring no backend para gerir a lógica de negócio e as operações fundamentais (CRUD) sobre uma base de dados PostgreSQL.

O foco principal foi a criação de um sistema organizado e eficiente, capaz de lidar com consultas estruturadas e persistência de dados crítica para o domínio da limpeza urbana. Para garantir a portabilidade e a facilidade de configuração em qualquer máquina, a infraestrutura foi totalmente orquestrada com Docker Compose, integrando o servidor de base de dados, a interface de gestão (PgAdmin) e a aplicação num único fluxo de trabalho automatizado.

Principais Aspetos Técnicos

Modelagem PostgreSQL

Implementação de uma estrutura relacional robusta no PostgreSQL, focada na normalização de dados, integridade referencial e criação de consultas SQL otimizadas para o domínio do projeto.

Integração Java Spring

Desenvolvimento de uma interface em Java Spring para a orquestração das operações de CRUD. Aplicação de boas práticas de programação para garantir uma comunicação fluida entre o código e a base de dados.

Ambiente Conteinerizado

Utilização de Docker e Docker Compose para isolar e orquestrar a aplicação e o banco de dados. Esta abordagem garante que o sistema funcione de forma idêntica em qualquer ambiente de desenvolvimento.

Domínio de Limpeza Urbana

Aplicação de regras de negócio específicas para a gestão urbana, organizando entidades e processos que simulam desafios reais de manutenção e organização de serviços municipais.
